# 한 끗 가이드 — 프론트 통합 안내 (SCRUM-94)

브랜치: `feature/SCRUM-94-one-touch-guide`

프론트(React) 쪽 UI/흐름은 woz 프로토타입과 디자인 시안 기준으로 구현해 두었습니다.
**백엔드가 붙으면 동작하도록 “준비된” 상태**이며, 실제로 가이드가 뜨려면 아래 *백엔드 계약* 부분을 맞춰주시면 됩니다.

> ⚠️ **전제**: 이 구현은 전달해주신 **`woz.zip`(GuideEntry/GuideMessage/NextSteps/Roadmap/GuideAsset) 프로토타입과 디자인 시안**을 바탕으로 만들었습니다.
> 입력 폼·5섹션 구조·레퍼런스 반응·성장 흐름·보조 API(`/guide`,`/roadmap`,`/adopt`,`/image`,`/guide-asset`)·필드명(`blocks`,`next_steps`,`primary_focus` 등) 모두 woz 기준 **가정값**입니다.
> 아직 백엔드와 통합 전이라, **실제 통합 과정에서 응답 형태·필드명·엔드포인트가 달라지면 그에 맞춰 바꿔서 진행하면 됩니다.** 바뀔 가능성이 있는 지점은 코드에 `// TODO(백엔드 통합)` 주석으로, 매핑은 `extractGuide` 한 곳에 모아뒀습니다.

---

## 1. 전체 흐름

1. 채팅 입력바의 **첨부(클립) 버튼** → `GuideRequestModal`(한 끗 가이드 입력 폼)이 열림.
2. 모달에서 **그림 + 고민 메시지 + 칩 + 화풍(medium) + 상태(intent)** 입력 → “가이드 요청하기”.
3. 이미지 업로드(`/images/upload`) 후 **기존 채팅 전송(`POST /projects/{id}/chat`)** 에 실어 보냄.
   - 별도 `/guide` 호출 없음. **백엔드가 첨부 이미지로 의도를 분류해 가이드를 만들어 응답**한다는 전제.
4. 채팅 응답이 “가이드”면 → 메시지에 **가이드 카드**(“○○ · 한 끗 가이드 보기”) 노출.
5. 카드 클릭 → 좌측 보드 자리에 **전체 가이드 패널**(`GuidePanel`) 표시.

## 2. 백엔드가 맞춰줘야 하는 계약 ⭐

### 2-1. `POST /projects/{id}/chat` 요청 (프론트가 추가로 보냄)
기존 body에 아래 두 필드가 **선택적으로** 추가됩니다(가이드 모달에서 온 경우만):
```jsonc
{
  "message": "…",
  "sessionId": "…",
  "imageUrl": "업로드된 이미지 URL",
  "intent": "open" | "finished",   // 작업중 / 완성작
  "medium": "" | "sketch"          // 자동 / 스케치
}
```
> 백엔드가 무시해도 무방. 의도 분류에 활용하면 됨.

### 2-2. `/chat` 응답 — 가이드일 때
프론트는 `src/pages/chat/ChatPage.jsx`의 **`extractGuide(res)`** 에서 가이드를 꺼냅니다.
현재 가정한 형태(woz 기준, **필드명 확정 시 이 함수만 수정**하면 됨):

```jsonc
{
  "message": "채팅 말풍선 텍스트",
  // 아래 둘 중 하나로 가이드 신호:
  "mode": "coach",          // 또는
  "guide": {                // res.guide 로 감싸도 됨
    "guide_id": "…",
    "primary_focus": "value_structure",   // 축(sub_problem) id
    "title": "명암 대비",                  // 없으면 primary_focus 라벨 사용
    "summary": "…",                        // (선택) 축 없을 때 상단 요약
    "recommend_practice": "…",             // (선택) 축 없을 때 '추천 연습' 본문
    "one_thing": "…",
    "blocks": [{
      "observation": "1. 분석 본문",
      "effect": "2. 읽히는 느낌 본문",
      "direction": "3. 한 끗 포인트 본문",
      "guide_asset": { "ref_id": "…", "label": "…", "caption": "…" },
      "reference_ids": ["id1","id2","id3"]
    }],
    "next_steps": {
      "focus": "value_structure",
      "focus_practice": "…",
      "next_goal": "weight_balance",
      "next_goal_practice": "…"
    }
  }
}
```

- **일반(축 있음)**: 1.분석 / 2.읽히는 느낌 / 3.한 끗 포인트 / 4.추천 레퍼런스 / 5.앞으로 해야 할 것 / 성장 흐름
- **축 없을 때**: `summary` + `recommend_practice` 가 오면 → 1.추천 연습 / 2.한 끗 포인트 / 3.추천 레퍼런스 / 4.앞으로 해야 할 것 / 성장 흐름 (분석·읽히는 느낌 생략, 번호 당김)
  - ⚠️ `summary`/`recommend_practice` 의 **실제 필드명은 미확정** — 백엔드 형태에 맞춰 `extractGuide` 매핑만 조정.

### 2-3. 보조 리소스 — `src/pages/chat/guideApi.js`
환경변수 **`VITE_GUIDE_API_BASE`** 로 베이스 주소를 받습니다(비어 있으면 호출 스킵 → 화면 안 깨짐). woz(artcoach) 기준 경로:

| 용도 | 호출 |
|------|------|
| 성장 흐름 | `GET {BASE}/roadmap?user_id=` |
| 레퍼런스 반응(👍/👎) | `POST {BASE}/adopt` `{ guide_id, reference_id, event:"liked"\|"disliked" }` |
| 레퍼런스 이미지 | `GET {BASE}/image/{id}` (ref에 `url`이 있으면 그걸 우선 사용) |
| 한 끗 포인트 도식 | `GET {BASE}/guide-asset/{ref_id}` |

> 우리 백엔드(8080)가 프록시할지, 별도 FastAPI(8000)를 직접 칠지에 따라 BASE/경로를 조정하세요.

## 5. 데모(임시) — 백엔드 없이 화면 확인용

- 채팅 URL에 **`?guideDemo=1`** 추가 → 가짜 가이드 메시지 주입(무게중심·축 없을 때 변형).
  예) `http://localhost:5173/projects/<id>/chat?guideDemo=1`
- 모든 데모 코드는 `ChatPage.jsx`에 **`DEMO(임시)` 주석**으로 분리, 별도 커밋(`SCRUM-94: [임시] …`)에 격리.
- **통합 후 해당 커밋만 revert**하면 데모 깔끔히 제거됨.
