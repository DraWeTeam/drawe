# 4. User Interface Prototype

> UI 프로토타입은 **Figma**로 관리한다. UX/UI 와이어프레임과 **Design System**을 디자이너가 별도로 작업 중이며, **Figma 프로토타입 링크는 추후 첨부(개별 제출)** 예정이다. 아래는 주요 화면과 인터랙션 의도다.

## 4.1 Figma 프로토타입

📐 **Figma 프로토타입(UX/UI 와이어프레임 · Design System) — 링크 추후 첨부 예정.**

## 4.2 주요 화면

| 화면 | 설명 |
|---|---|
| **로그인 / 회원가입 / 약관동의** | 이메일 로그인·가입 / Google OAuth 진입 / 약관동의(consent) |
| **온보딩** | 가입 후 취향 태그 선택(추천 랭킹 soft boost용). *현 빌드에서 라우트 비활성(설계상 존재)* |
| **프로젝트 목록** | 카드 그리드, 정렬(최근/생성/이름)·검색·상태 필터 |
| **새 프로젝트** | 주제·기법·분위기 입력 |
| **레퍼런스 보드**(`/projects/:id/board`) | 키워드 검색 + 좋아요/싫어요 피드백 + AI 생성 — 실제 레퍼런스 추천 1차 UI |
| **채팅(레퍼런스 추천)**(`/projects/:id/chat`) | 레퍼런스 그리드(`[N]`), 핀, 미술 조언, 검색 0건 시 생성 제안 |
| **갤러리 / 아카이브** | 완성작(내 그림·COMPLETED 프로젝트) · 완성작 상세(회고) · 프로젝트별 레퍼런스 아카이브 |
| **설정** | 설정·요금제(plan)·약관/정책(policy) |

> **가이드**는 독립 화면이 아니라 **레퍼런스 보드·채팅에 통합**된다(별도 `pages/guide` 없음 — `board/BoardGuideChat`, `chat/GuideModal`·`GuideForm`). 그림 업로드 → 진단 카드(구도·명암 등)·코칭·툴팁은 이 통합 채팅 안에서 제공된다.

## 4.3 인터랙션 포인트
- **레퍼런스 그리드**: 검색 결과 `[N]` 번호 ↔ 응답 본문 인용 일치. 핀 시 "고정 N" 슬롯(최대 3).
- **멀티턴**: 이어묻기(KEEP/FOLLOWUP)는 직전 레퍼런스를 재사용, 새 검색(NEW_SEARCH)만 그리드 갱신.
- **생성 제안**: 검색 0건이면 "AI 이미지로 생성해드릴까요?" 버튼 노출 → 수락 시 생성.
- **가이드 툴팁**: 진단 축(구도·명암·색감 등)에 대한 설명 오버레이.
