# 1. Introduction

## 1.1 목적
본 문서는 **DraWe**(AI 그림 레퍼런스 추천·드로잉 가이드 어시스턴트)의 소프트웨어 설계를 기술한다.
시스템 구조, AI 파이프라인, 도메인 모델, 주요 흐름, 데이터·배포 설계를 정의해 개발·평가·유지보수의 기준을 제공한다.

## 1.2 범위

**In Scope**
- 자연어 요청 기반 **레퍼런스 이미지 추천** (검색 + 하이브리드 re-rank)
- **멀티턴 대화** 기반 미술 조언 (구도·명암·색감·기법)
- 업로드 그림 **비전 진단·코칭**(가이드) + 성장(growth)
- **프로젝트**(주제·기법·분위기·핀) 관리, 정렬·검색
- **갤러리/아카이브**(완성작·레퍼런스), **AI 이미지 생성**
- Google OAuth 인증, 관리자 대시보드, 분석 이벤트
- **온보딩 취향 개인화** — 추천 랭킹 soft boost + "취향 결" 표시(연습 목적 적합성 우선, 결이 맞는 것 우선)
- **보드 피드백 반영** — 싫어요=다음 검색에서 제외(+3회 누적 시 생성 유도 모달), 좋아요=반응 저장(정렬·유지용, 검색 랭킹 무반영)

**Out of Scope (현 단계)**
- 가이드(👍/👎) 피드백의 **모델 재학습 반영** — 표시·저장까지이며, 재학습 파이프라인은 미구현

## 1.3 시스템 개요
DraWe는 **프론트(React) — 백엔드(Spring Boot) — AI 서비스(FastAPI ×2) — 데이터스토어(MySQL·Redis·Pinecone(채팅·보드 검색)·Qdrant(가이드 코퍼스))** 로 구성된 멀티서비스 시스템이다.
백엔드가 도메인 로직과 외부 연동을 오케스트레이션하고, CLIP 임베딩·벡터 검색·LLM 합성을 거쳐 레퍼런스와 미술 조언을 생성한다.
(전체 토폴로지·파이프라인 그림은 [README](./README.md) 참고. 협업·배포 프로세스 요약은 [collab-process](./img/collab-process.svg) — Jira→PR→CI/리뷰→머지→CD→ArgoCD→EKS.)

## 1.4 주요 기능 (전체)

| 도메인 | 기능 |
|---|---|
| **인증(auth)** | Google OAuth 로그인, JWT(access/refresh) 발급·갱신, 로그아웃 |
| **프로젝트(project)** | 생성, 목록(정렬·검색·상태필터·페이징), 상세, 수정, 삭제 |
| **핀/레퍼런스** | 핀 추가·해제·조회(슬롯 3), 프로젝트 레퍼런스 |
| **이미지(image)** | 업로드, 서빙, **AI 이미지 생성(Bedrock Stability)** |
| **검색(search)** | CLIP 임베딩 → Pinecone 벡터검색 → MySQL 메타 → **태그 IDF re-rank** |
| **채팅/LLM(llm)** | 의도 분류 → 검색/유지/합성(설계상 워크플로), 멀티턴, 초기화, **자기비평(이미지)**, 이미지 생성 |
| **가이드(guide)** | 그림 업로드 진단·코칭, 가이드/레퍼런스 피드백, 성장(growth) |
| **갤러리(gallery)** | 완성작 갤러리(내 그림·COMPLETED 프로젝트), 완성작 상세(회고), 레퍼런스 아카이브(프로젝트별) |
| **관리/분석** | 관리자 대시보드, 분석 이벤트(GA4·내부 메트릭) |

> **의도→라우팅→COMPOSE 워크플로는 미채택(retired)** — 제품 방향이 무드보드 검색+가이드로 정리되며 채택되지 않았다. `IntentCode`(13종)는 폐기된 설계 카탈로그(historical)이며, prod live-intents 는 비활성이고 코드는 dormant(`WorkflowComposeProperties.liveIntents` 기본 `noneOf`)로 잔존한다. 채팅 응답은 여전히 **레거시 직접 합성** 경로가 담당한다. 위 표의 "의도 분류 → 검색/유지/합성"은 폐기된 설계 목록이며, 활성 런타임과 구분해 읽는다.
