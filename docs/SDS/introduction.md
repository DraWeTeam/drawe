# 1. Introduction

## 1.1 목적
본 문서는 **DraWe**(AI 그림 레퍼런스 추천·드로잉 가이드 어시스턴트)의 소프트웨어 설계를 기술한다.
시스템 구조, AI 파이프라인, 도메인 모델, 주요 흐름, 데이터·배포 설계를 정의해 개발·평가·유지보수의 기준을 제공한다.

## 1.2 범위

**In Scope**
- 자연어 요청 기반 **레퍼런스 이미지 추천** (검색 + 하이브리드 re-rank)
- **멀티턴 대화** 기반 미술 조언 (구도·명암·색감·기법)
- 업로드 그림 **비전 진단·코칭**(가이드) + 성장(growth)
- **프로젝트**(주제·기법·분위기·핀) 관리, 정렬·검색
- **갤러리/아카이브**(완성작·레퍼런스), **AI 이미지 생성**
- Google OAuth 인증, 관리자 대시보드, 분석 이벤트

**Out of Scope (현 단계)**
- 온보딩 기반 개인화(프론트 차단) — 추후 growth/피드백 기반으로 대체 예정
- 좋아요/싫어요 피드백의 추천 반영(현재 수집만)

## 1.3 시스템 개요
DraWe는 **프론트(React) — 백엔드(Spring Boot) — AI 서비스(FastAPI ×2) — 데이터스토어(MySQL·Redis·Pinecone)** 로 구성된 멀티서비스 시스템이다.
백엔드가 도메인 로직과 외부 연동을 오케스트레이션하고, CLIP 임베딩·벡터 검색·LLM 합성을 거쳐 레퍼런스와 미술 조언을 생성한다.
(전체 토폴로지·파이프라인 그림은 [README](./README.md) 참고)

## 1.4 주요 기능 (전체)

| 도메인 | 기능 |
|---|---|
| **인증(auth)** | Google OAuth 로그인, JWT(access/refresh) 발급·갱신, 로그아웃 |
| **프로젝트(project)** | 생성, 목록(정렬·검색·상태필터·페이징), 상세, 수정, 삭제 |
| **핀/레퍼런스** | 핀 추가·해제·조회(슬롯 3), 프로젝트 레퍼런스 |
| **이미지(image)** | 업로드, 서빙, **AI 이미지 생성(Bedrock Stability)** |
| **검색(search)** | CLIP 임베딩 → Pinecone 벡터검색 → MySQL 메타 → **태그 IDF re-rank** |
| **채팅/LLM(llm)** | 의도 분류 → 검색/유지/합성, 멀티턴, 초기화, **자기비평(이미지)**, 이미지 생성 |
| **가이드(guide)** | 그림 업로드 진단·코칭, 가이드/레퍼런스 피드백, 성장(growth) |
| **갤러리(gallery)** | 완성작 갤러리(AI 이미지), 레퍼런스 아카이브(프로젝트별) |
| **관리/분석** | 관리자 대시보드, 분석 이벤트(GA4·내부 메트릭) |
