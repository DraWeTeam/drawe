# Figma 스펙 컴펜디엄 (UX-UI-Wireframe)

> **화면 작업 선행 참조 문서.** 프론트/백엔드에서 특정 화면(SCR-*)을 구현·수정하기 전에, 이 문서에서 해당 섹션의 스펙(화면 목적·상태·동작·연동·정본 표/매트릭스)을 먼저 확인한다. Figma 정본과 코드가 어긋나면 정본을 따르되, 전제 불일치는 보고한다.

- **원본**: Figma `UX-UI-Wireframe` (fileKey `vEWEwSklFvrk1wOzhtHsOX`), 단일 캔버스에 섹션 20개.
- **수집 범위**: 각 섹션에서 **화면 프레임(SCR-*) 바깥에 배치된 스펙 자료**만 수집 — (a) `description` 캡션 카드, (b) `[진입·이탈]/[상태]/[동작]/[연동]` 태그형 텍스트 노트, (c) `image NNN` 스펙 이미지(표·매트릭스·목업). 화면 내부 UI, 플로우 커넥터 화살표, 떠 있는 모달/메뉴 인스턴스는 스펙이 아니므로 제외.
- **전사 원칙**: 한국어 verbatim(오탈자 미보정). 표/매트릭스는 마크다운 표로. 도면에서 잘린 원문은 잘린 그대로 표기.
- **수집일 기준**: feature/image-guiding 브랜치 작업 중 전수 스윕(2026-07-05). 섹션 목록은 프레임 외 스펙 유무와 무관하게 전 섹션 수록.
