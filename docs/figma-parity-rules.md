# Figma 정합(UI Parity) 작업 규칙 — 팀 공유 정본

> Figma 디자인을 구현·수정할 때 반드시 따른다. "정합 완료" 주장은 아래 ④의 대조표 없이는 성립하지 않는다.
> (SCRUM-107 UI 정합 작업부터 적용. 로컬 `CLAUDE.md`는 이 문서를 참조만 한다 — 규칙 정본은 여기.)

## 원칙

### ① 눈대중 금지 — 실측이 첫 단계
구현/수정 전에 **해당 프레임의 실측값을 표로 추출**한다. 도구:
- `get_metadata(fileKey, nodeId)` — 구조·위치·크기(px) 개요.
- `get_design_context(fileKey, nodeId)` — 색(hex)·radius·간격·폰트(크기/굵기/행간) 등 상세 스타일.

추출 표 최소 컬럼: `요소 | Figma 값(px/hex/radius/간격/폰트) | 근거 노드ID`.

### ② 구현은 표의 값 문자 그대로
- 표에 적힌 값을 **그대로** 반영한다(반올림·근사 금지).
- 디자인 토큰(`:root` CSS 변수)에 이미 있는 값은 **토큰을 참조**한다(하드코딩 대신).

### ③ 검증 = 실렌더 vs Figma export + 픽셀 대조
- **실렌더 스크린샷** 과 **Figma export(get_screenshot)** 를 나란히 비교.
- 주요 치수 **3~5개를 픽셀 단위로 대조**(폭·높이·간격·폰트크기·radius 등).

### ④ "정합 완료" = 대조표가 증거
정합 완료를 주장하는 리포트에는 반드시 다음 대조표를 붙인다:

| 요소 | Figma 값 | 구현 값 | 일치 |
|---|---|---|---|

## 프로세스(화면 단위)
1. 실측표 추출(①) — ★compendium·annotation 해당 섹션 선행 참조.
2. 현 구현 실렌더 대조 → `[화면/요소/Figma값/구현값/차이]` 갭 표 + 스크린샷 나란히.
3. 갭 표 기준으로 값 그대로 수정(②) — **화면 단위 커밋**.
4. 커밋마다 ③ 검증 + 회귀(주변 화면·/chat·해당 기능) 0 + `eslint`·`build` 통과.
5. ★사용자 최종 게이트: 스크린샷 세트를 사용자가 Figma와 눈으로 비교 → 통과 화면만 PR 범위 확정.
